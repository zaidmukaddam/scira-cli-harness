# Scira CLI Harness

Headless, Flue-powered research CLI: search (Parallel, Exa, Firecrawl), link fetching, optional synthesis, Daytona sandboxing, and a built-in benchmark suite. It is designed to evolve independently of a web UI while sharing similar behavior to a Scira-style product.

- **CLI entry:** `src/cli.ts` (wrapper around the Flue agent)
- **Agent:** `.flue/agents/research.ts`
- **Role / system prompt:** `.flue/roles/researcher.md`

## Requirements

- [Bun](https://bun.sh/) 1.x or newer

## Quick start

### 1. Install

```bash
git clone https://github.com/zaidmukaddam/scira-cli-harness.git
cd scira-cli-harness
bun install
```

### 2. Environment

```bash
cp .env.example .env.local
```

Configure at least:

- **Search:** one of `PARALLEL_API_KEY`, `EXA_API_KEY`, or `FIRECRAWL_API_KEY`
- **Synthesis:** the key for your `--model` provider (for example `OPENAI_API_KEY`, `XAI_API_KEY`, `ANTHROPIC_API_KEY`, or `OPENROUTER_API_KEY`)
- **Daytona (optional):** `DAYTONA_API_KEY` for sandbox runs

**Precedence:** shell environment wins, then `SCIRA_ENV_FILE` (absolute path to an env file loaded last among files), then repo `.env` / `.env.local`. Optionally set `SCIRA_PROJECT_ROOT` to an absolute path of another project whose `.env` / `.env.local` are loaded first (lowest precedence among files), so you can reuse a sibling app’s secrets without copying them.

### 3. Build

```bash
bun run build
bun run build:flue
```

### 4. Smoke test

Source gathering only:

```bash
bun run src/cli.ts "what is the Flue framework?" --provider parallel --no-synthesis
```

Full Flue synthesis:

```bash
bun run src/cli.ts research "what is the Flue framework?" --provider parallel --model openai/gpt-4.1-mini
```

Daytona:

```bash
bun run src/cli.ts sandbox --code "print(2 + 2)"
```

## Usage

```bash
bun run src/cli.ts "what is possible with the Flue framework?" --provider parallel --no-synthesis
bun run src/cli.ts research "compare Scira and Perplexity" --model openai/gpt-4.1-mini
bun run src/cli.ts research "analyze these benchmark numbers" --model openai/gpt-4.1-mini --sandbox
bun run src/cli.ts benchmark --providers parallel,exa --models openai/gpt-4.1-mini --limit 2
bun run src/cli.ts sandbox --code "print(2 + 2)"
bun run src/cli.ts sandbox --file ./analysis.py --install pandas,numpy
bun run src/cli.ts "latest AI search products" --topic news --start-date 2026-01-01 --json
```

After `bun run build`:

```bash
bun run dist/scira-harness.js "latest AI search products" --no-synthesis
```

Invoke the Flue agent directly:

```bash
bun run run:flue -- \
  --payload '{"query":"compare Scira and Perplexity","maxResults":5,"topic":"general","quality":"default","synthesize":true,"model":"openai/gpt-4.1-mini"}'
```

## Benchmarking

Suite definition: `benchmarks/research-smoke.json`. Nineteen source-backed checks cover facts, domains, citations, numeric reasoning, caveats, negative controls, and `fetch_links` when required.

```bash
bun run src/cli.ts benchmark --list-cases
bun run src/cli.ts benchmark \
  --providers parallel,exa \
  --models openai/gpt-4.1-mini,anthropic/claude-sonnet-4-6 \
  --limit 2 \
  --concurrency 2
bun run src/cli.ts benchmark --list-models --model-provider openai --configured-only
bun run src/cli.ts benchmark \
  --providers parallel \
  --models openai/gpt-4.1-mini \
  --json \
  --out benchmark-results/run.json
```

Useful flags: `--suite`, `--case`, `--limit`, `--concurrency`, `--quiet`, `BENCHMARK_MODELS`, `BENCHMARK_CONCURRENCY`.

Write JSON or Markdown exports to **`benchmark-results/`** (gitignored scratch; create it if you like) so ad-hoc runs do not clutter the tree. **Canonical logs** that back blog posts or releases should live under **`docs/benchmark-runs/`** and are committed.

Cases are inspired by SimpleQA, BrowseComp, and GAIA-style methodology; they are local harness fixtures, not copied hidden eval sets.

## Benchmark chart visuals (optional)

PNG charts from a Markdown results file are generated by a Python script run with [uv](https://docs.astral.sh/uv/):

```bash
uv run visualize_benchmark.py
```

Outputs go to `benchmark_charts/` (gitignored). Ensure `exa.svg` is present in the repo root for the Exa logo in charts.

## Sources and link fetching

Synthesized Markdown includes a **Source Ledger** from the run. Inside the agent, `fetch_links` can use Parallel Extract, Exa `getContents`, or Firecrawl `scrape`.
